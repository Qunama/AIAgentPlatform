AIAgentPlatform — AI-агент для работы с проектами (.NET 8)

МАНИФЕСТ ПРОЕКТА

МИССИЯ: Создать AI-агента на .NET 8 с локальной LLM, который получает запросы на естественном языке, самостоятельно вносит правки в .NET проект, ведёт и читает документацию, понимает общую картину работы.

ТЕКУЩАЯ СТАДИЯ: Этапы 1-8 завершены. Проект готов к использованию.

КОМПОНЕНТЫ СИСТЕМЫ

АГЕНТЫ (4):
- OrchestratorAgent — главный оркестратор. Принимает запрос, получает контекст от ContextAgent, строит системный промпт с {TOOLS}, вызывает LLM, парсит JSON-ответы (ExtractActionPlanRobust), вызывает инструменты через ToolRegistry.ExecuteToolAsync. Цикл: Explore -> Read -> Write -> Validate -> Document -> Report. Защита от зацикливания (MaxIterations=10). Инжектит: ToolRegistry, ILlmService, ContextAgent, BuildValidationService, ReportService, MetricsService.
- CodeEditorAgent — чтение/изменение файлов. Инструменты: ReadFileTool, WriteFileTool.
- DocumentationAgent — работа с документацией. Инструменты: ReadDocumentationTool, UpdateDocumentationTool.
- ContextAgent — управление контекстом (ProjectContext). Методы: SetProjectPath, CacheProjectStructure, RegisterFileModification, RegisterError, RecordInteraction, SetPhase, GetContextSummary, ResetErrors.

ИНСТРУМЕНТЫ (7, все реализуют IAgentTool, регистрируются через [AgentTool] и AddAgentTools):
- read_file — чтение файла. Зависимости: IOptions<AgentOptions>.
- write_file — запись файла с созданием директорий. Зависимости: IOptions<AgentOptions>.
- project_structure — сканирование структуры. Зависимости: IOptions<AgentOptions>.
- read_documentation — семантический поиск по Qdrant. Зависимости: IOllamaEmbeddingClient, IQdrantService, IOptions<QdrantOptions>.
- update_documentation — обновление/создание секций с реиндексацией. Зависимости: IDocumentationService, IOllamaEmbeddingClient, IQdrantService, IOptions<AgentOptions>, IOptions<QdrantOptions>.
- run_shell_command — выполнение команд (белый список: dotnet, git, npm, docker, echo, dir, ls, cat, type). Зависимости: CliWrap, IOptions<AgentOptions>.
- git_diff — Git status/diff/log. Зависимости: LibGit2Sharp, IOptions<AgentOptions>.

ФОНОВЫЕ СЛУЖБЫ (4):
- AgentWorkerService (HostedService) — интерактивный цикл, выбор режима A/T.
- DocumentationIndexerService (HostedService) — индексация документации при старте.
- DocumentationWatcherService (HostedService) — отслеживание изменений .md/.txt через FileSystemWatcher.
- BuildValidationService (встроен в OrchestratorAgent) — dotnet build с ретраями.

СЕРВИСЫ:
- ILlmService / LlmService — вызов Ollama /api/generate с Polly-ретраями.
- LlmResponseParser — статический хелпер: ExtractCodeBlock, ExtractActionPlan, ExtractJson, ConvertJsonElement.
- IOllamaApiClient / OllamaApiClientWrapper — HTTP-клиент для /api/generate.
- IOllamaEmbeddingClient / OllamaEmbeddingClient — HTTP-клиент для /api/embeddings.
- IDocumentationService / DocumentationService — поиск .md/.txt, чанки, векторизация.
- IQdrantService / QdrantService — Qdrant REST API.
- BuildValidationService — dotnet build, BuildResult, BuildAttempt, ExtractBuildErrors.
- ReportService — GenerateReport (полный), GenerateShortReport (краткий).
- MetricsService — агрегация RequestMetrics, GetSummary().
- PromptLoader — загрузка .txt промптов с кешированием.
- PromptTesterService — ручное тестирование промптов.
- CliService — CLI-команды.
- SignalRLoggingService — отправка событий через AgentHub.

ТИПИЧНЫЙ WORKFLOW ОРКЕСТРАЦИИ

1. Пользователь отправляет запрос: "Добавь метод Multiply в Calculator".
2. ContextAgent предоставляет сводку контекста (ProjectContext.GetContextSummary).
3. OrchestratorAgent формирует системный промпт с {TOOLS} и контекстом.
4. LLM возвращает JSON: {"action":"delegate","tool":"project_structure","args":{"path":"."}}.
5. ExtractActionPlanRobust парсит JSON, ExecuteToolFromPlan вызывает инструмент.
6. Результат добавляется в conversationHistory (StringBuilder).
7. Цикл повторяется: read_file -> write_file -> run_shell_command (dotnet build).
8. BuildValidationService выполняет сборку, при ошибке — до 3 ретраев.
9. При успехе: git_diff -> report.
10. ContextAgent записывает взаимодействие в историю.
11. MetricsService записывает метрики запроса.
12. ReportService генерирует форматированный отчёт.

ВАЖНЫЕ ЗАМЕТКИ

- Модель qwen2.5-coder:7b-instruct требует ~8 ГБ RAM. Можно заменить в appsettings.json.
- Qdrant: вектор 3584 (embedding_length модели), Cosine distance.
- Semantic Kernel используется только для Memory.
- Microsoft.Agents.Hosting исключён — собственная оркестрация.
- Dockerfile в корне проекта (рядом с .sln), не в ProjectAIAgent.Host/.
- Для CI/CD нужны секреты DOCKER_USERNAME и DOCKER_PASSWORD в GitHub.
- Все изменения кода через Git (LibGit2Sharp).
- Результаты read_file обрезаются до 3000 символов (защита контекстного окна).
- RunShellCommandTool: белый список команд, запрет rm/del/format.
- DocumentationAgent в тестовом режиме честно отказывается документировать без кода.
- PromptTesterService: /role для смены роли, /system для просмотра промпта.
- FileSystemWatcher: дебаунс 3 секунды, отключение через WatchDocumentation: false.
- MetricsService: метрики доступны через API (status) и CLI (status).
- Все инструменты регистрируются автоматически — достаточно добавить класс с [AgentTool] и IAgentTool.
