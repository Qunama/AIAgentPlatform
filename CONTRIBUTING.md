# Руководство контрибьютора

## Как помочь проекту

1. Форкните репозиторий
2. Создайте ветку для фичи/исправления: `git checkout -b feature/название`
3. Внесите изменения
4. Запустите тесты: `dotnet test`
5. Создайте коммит: `git commit -m "Описание изменений"`
6. Отправьте в форк: `git push origin feature/название`
7. Создайте Pull Request

## Правила именования коммитов

- `Этап X, Подзадача Y: Краткое описание`
- Или на английском: `Stage X, Subtask Y: Brief description`

## Структура проекта

- `ProjectAIAgent.Host/` — точка входа, API, CLI, Web UI
- `ProjectAIAgent.Core/` — агенты, инструменты, сервисы
- `tests/` — юнит-тесты и песочница
- `local-infra/` — Docker Compose для Ollama и Qdrant

## Добавление нового инструмента

1. Создайте класс в `Core/Tools/`, реализующий `IAgentTool`
2. Добавьте атрибут `[AgentTool("имя", "описание")]`
3. Инструмент зарегистрируется автоматически через `AddAgentTools`

## Запуск тестов

```bash
dotnet test
```bash

30 юнит-тестов для инструментов и сервисов.

## Требования к коду

- C# 12, .NET 8
- XML-документация для публичных методов
- Юнит-тесты для новых инструментов и сервисов
- 0 предупреждений сборки
